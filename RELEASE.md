# Release Guide

TokenScope uses GitHub Actions and `tauri-apps/tauri-action` to publish unsigned macOS and Windows desktop builds to draft GitHub Releases.

## Create A Tag

Update versions first if needed, then create and push a release tag:

```bash
git tag v0.1.0
git push origin v0.1.0
```

Pushing a `v*` tag triggers `.github/workflows/release.yml`. The workflow creates a draft release so maintainers can inspect artifacts before publishing.

## Trigger Manually

You can also run the `Release` workflow from the GitHub Actions UI with `workflow_dispatch`.

Use the `tag_name` input, for example:

```text
v0.1.0
```

## Download Artifacts

After the workflow completes, open the GitHub Release for the tag.

- macOS users download the `.dmg`. The workflow also builds the macOS app bundle.
- Windows users download the `.msi` or `.exe` installer generated by MSI/NSIS.
- The release remains a draft until a maintainer reviews and publishes it.

## Unsigned Build Prompts

Current MVP builds are unsigned.

- macOS may show Gatekeeper warnings because the app is not Developer ID signed or notarized.
- Windows may show SmartScreen warnings because the installer is not code signed.

Users should only install builds downloaded from the project GitHub Releases page.

## Local Smoke Test Checklist

After building locally:

- Open TokenScope.app.
- Confirm Dashboard renders.
- Confirm About shows version 0.1.0.
- Confirm Sources page renders.
- Confirm Sessions filters render.
- Confirm Settings privacy section renders.
- Confirm CSV import preview opens or fails gracefully.
- Confirm no obvious runtime console errors.

## Future Signing Setup

Before a production release:

1. Add Apple Developer ID certificate secrets.
2. Configure macOS hardened runtime signing and macOS notarization.
3. Add Windows code signing certificate secrets.
4. Add timestamp server configuration for Windows signing.
5. Re-run release builds and verify the downloaded installers on clean machines.
